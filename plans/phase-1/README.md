# Phase 1: End-to-End Proof

**Goal:** `loom build` takes TOML manifests → resolves dependencies → assembles file-set → generates Vivado Tcl → runs `vivado -mode batch` → reports pass/fail.

**Validation criterion (spec §15, Phase 1):** An existing Vivado project, re-expressed as Loom manifests, builds successfully.

**What is explicitly OUT of Phase 1:**
- No generators (no GENERATE step — goes straight RESOLVE → ASSEMBLE)
- No platforms (`[target]` block in project.toml instead)
- No profiles, variants, OOC synthesis
- No Python plugin loading (Vivado backend is Rust)
- No metrics extraction (just exit code + log path)
- No constraint templating
- No `--resume`, `--stop-after`, `--dry-run`
- `-j` flag parsed but ignored

## Task Execution Order

Tasks must be completed in this order (each depends on the previous):

| # | Task | Key Deliverable |
|---|---|---|
| 01 | [Project Scaffold](./01-project-scaffold.md) | Cargo workspace builds |
| 02 | [Component Manifest](./02-manifest-component.md) | `component.toml` parsed |
| 03 | [Project & Workspace Manifests](./03-manifest-project-workspace.md) | `project.toml`, `workspace.toml` parsed |
| 04 | [Workspace Discovery](./04-workspace-discovery.md) | Find workspace root and members |
| 05 | [Dependency Resolver](./05-dep-resolver.md) | Dependency graph built, cycles detected |
| 06 | [Lockfile](./06-lockfile.md) | `loom.lock` generated and validated |
| 07 | [File-Set Assembly](./07-fileset-assembly.md) | `AssembledFilesets` produced |
| 08 | [Plugin Traits](./08-plugin-traits.md) | `BackendPlugin` Rust trait defined |
| 09 | [Pre-Build Validation](./09-prebuilt-validation.md) | Phase 4 checks pass/fail correctly |
| 10 | [Vivado Tcl Generator](./10-vivado-tcl-gen.md) | Valid Tcl script produced |
| 11 | [Vivado Executor](./11-vivado-executor.md) | `vivado -mode batch` invoked, output captured |
| 12 | [Vivado Env Check](./12-vivado-env-check.md) | Vivado installation detected and verified |
| 13 | [Error Types](./13-error-types.md) | `LoomError`, diagnostics, exit codes |
| 14 | [CLI Skeleton](./14-cli-skeleton.md) | `loom --help` works, commands dispatch |
| 15 | [CLI Commands](./15-cli-commands.md) | `build`, `clean`, `lint`, `deps tree`, `env check` work end-to-end |

## Test Fixtures Needed

Create these in `tests/fixtures/` before or alongside the tasks:

### `tests/fixtures/simple_project/`
```
workspace.toml
loom.lock             (generated by test)
lib/
  axi_common/
    component.toml
    rtl/
      axi_pkg.sv      (empty placeholder)
projects/
  my_design/
    project.toml
    src/
      top.sv          (empty placeholder)
    constraints/
      timing.xdc      (empty placeholder)
```

`workspace.toml`:
```toml
[workspace]
name = "test_workspace"
members = ["lib/*", "projects/*"]
```

`lib/axi_common/component.toml`:
```toml
[component]
name = "testorg/axi_common"
version = "1.0.0"

[filesets.synth]
files = ["rtl/axi_pkg.sv"]
```

`projects/my_design/project.toml`:
```toml
[project]
name = "my_design"
top_module = "top"

[target]
part = "xczu7ev-ffvc1156-2-e"
backend = "vivado"
version = "2023.2"

[filesets.synth]
files = ["src/top.sv"]
constraints = ["constraints/timing.xdc"]

[dependencies]
axi_common = ">=1.0.0"
```

### `tests/fixtures/multi_component/`
A workspace with a dependency chain: `project → comp_a → comp_b`. Used to test dependency graph ordering and circular dependency detection.
